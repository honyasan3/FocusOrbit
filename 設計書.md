# ソフトウェア詳細仕様書: FocusOrbit (Chrome Extension - Manifest V3)

## 1. システム要件とアーキテクチャ制約
生成AIへの指示前提として、以下のManifest V3 (MV3) の制約を厳格に守ること。
* **Service Workerのライフサイクル**: `background.js` は常駐せず、非アクティブ時にサスペンドされる。そのため、グローバル変数に状態を保持してはならない。すべての状態（State）は常に `chrome.storage.local` から読み書きすること。
* **DOMアクセスの禁止**: `background.js` 内で `document` や `window` オブジェクトにアクセスしないこと。
* **通信方式**: PopupやContent Scriptからの命令は、すべて `chrome.runtime.sendMessage` を使用し、`background.js` の `chrome.runtime.onMessage.addListener` で受け取るイベント駆動型とする。

## 2. データ構造仕様 (Data Schema)

### 2.1 Storage State (`chrome.storage.local`)
状態管理のズレを防ぐため、以下のJSONスキーマでデータを完全管理する。

~~~json
{
  "activeContextId": "context-work-123",
  "contexts": [
    {
      "id": "context-work-123",
      "name": "Work",
      "color": "blue",
      "tabs": [
        { "url": "https://...", "title": "...", "favIconUrl": "...", "pinned": false }
      ]
    }
  ],
  "archiveList": [
    { "id": "archive-999", "url": "https://...", "title": "...", "closedAt": 1714444800000 }
  ],
  "settings": {
    "autoArchiveMinutes": 60,
    "maxTabBudget": 15,
    "ignoreAudioTabs": true
  }
}
~~~

### 2.2 Message Passing API (ファイル間通信プロトコル)
UI (Popup/Content Script) と Background 間の通信は以下のActionで統一する。

* `SWITCH_CONTEXT`: `{ action: "SWITCH_CONTEXT", payload: { targetContextId: "..." } }`
* `CREATE_CONTEXT`: `{ action: "CREATE_CONTEXT", payload: { name: "..." } }`
* `RESTORE_ARCHIVE`: `{ action: "RESTORE_ARCHIVE", payload: { archiveId: "..." } }`
* `SEARCH_TABS`: `{ action: "SEARCH_TABS", payload: { query: "..." } }`

## 3. コアロジック詳細設計 (Core Logic Deep Dive)

### 3.1 コンテキスト切り替え (Identity Swapping) の厳密な手順
ウィンドウが閉じてしまうバグを防ぐため、以下の順番でAPIを実行すること。
1. **取得と除外**: `chrome.tabs.query({currentWindow: true})` で現在の全タブを取得。ただし `url` が `chrome://`, `edge://`, `about:` で始まるタブ、および `pinned: true` (ピン留め) のタブは保存・操作の対象外とする。
2. **状態保存**: 取得したタブ配列を `chrome.storage.local` の `activeContextId` の `tabs` に上書き保存。
3. **新規タブ生成**: 切り替え先のコンテキストの `tabs` データを読み込み、`chrome.tabs.create({ url: tab.url, active: false })` をループで実行。※アクティブにしないことで画面のチラつきを防ぐ。
4. **旧タブ削除**: (2)で取得した古いタブのIDリストを `chrome.tabs.remove([tabIds])` で一括削除。最後に新しいタブの先頭を `chrome.tabs.update({ active: true })` する。

### 3.2 自動アーカイブとタブ予算 (Smart Hibernation & Budgeting)
* **最終アクセス記録**: `chrome.tabs.onActivated` および `chrome.tabs.onUpdated` 発火時に、そのタブIDと現在時刻(`Date.now()`)をメモリマップ（またはStorage）に記録。
* **定期監視**: `chrome.alarms` で1分ごとに `checkIdleTabs` 関数を実行。
* **例外条件（重要）**: `chrome.tabs.query` で対象タブを検査する際、`audible: true` (音楽・動画再生中) のタブは、未アクセス時間が超過していても絶対にアーカイブしてはならない。
* **上限超過処理**: タブが新規作成された際 (`chrome.tabs.onCreated`)、現在のタブ数が `maxTabBudget` を超えた場合、最もアクセスが古いタブ（ピン留め・音声再生中を除く）を特定し、`archiveList` にURLを保存してから `chrome.tabs.remove` で閉じる。

## 4. UI/UX とエラーハンドリング仕様

### 4.1 エラーハンドリング (Error Handling)
* **Storage容量制限**: `archiveList` が肥大化してQuota超過エラーになるのを防ぐため、アーカイブ数が500件を超えたら古いものから自動削除（FIFO）するロジックをBackgroundに実装する。
* **無効なURL**: コマンドパレットからの復元時、URLが `undefined` や空文字の場合は処理をスキップし、ユーザーにエラーToastを表示する。

### 4.2 コマンドパレット (Command Palette)
* **マウント位置**: `content_scripts` を使用して、現在のウェブページのDOMに検索パレット（`<div id="focusorbit-palette-root">`）をShadow DOMでインジェクトする（既存サイトのCSSとの干渉を防ぐため）。
* **検索ロジック**: 正規表現を用いたあいまい検索（Fuzzy Search）を実装。タブの `title` と `url` の両方を検索対象とする。
