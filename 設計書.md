# ソフトウェア仕様書: FocusOrbit (Chrome Extension)

## 1. プロジェクト概要
* **名称**: FocusOrbit (フォーカス・オービット)
* **目的**: タブの乱立による「デジタル燃え尽き」を解消し、脳の認知負荷を最小化する。
* **コンセプト**: タブを管理するのではなく、ユーザーの「アイデンティティ（文脈）」を切り替える。

---

## 2. 解決すべき課題
1.  **タブ・ホーディング**: 情報を失う不安からタブを閉じられない。
2.  **コンテキストの喪失**: 仕事、趣味、家事のタブが混ざり、脳が疲弊する。
3.  **UIの不満**: 標準のタブスクロールが使いにくい。

---

## 3. 主要機能 (Functional Requirements)

### 3.1 Context Identity (コンテキスト分離)
* **Work / Life / Learning** などのグループを作成。
* グループを切り替えると、現在のタブセットを保存して閉じ、新しいグループのタブを展開する。
* `chrome.storage.local` を使用し、ブラウザを閉じても状態を維持する。

### 3.2 Smart Hibernation (スマート休止・アーカイブ)
* 一定時間（設定可能）アクティブでないタブを自動的に閉じる。
* 閉じたタブは「アーカイブリスト」に保存され、いつでも検索・復元が可能。
* これより「情報を失う不安」を解消し、メモリ消費を抑える。

### 3.3 Quick Search Palette (コマンドパレット)
* `Ctrl+K` (または `Cmd+K`) で起動。
* 開いているタブとアーカイブされたタブを横断して「あいまい検索」が可能。
* タブスクロールを不要にし、キーボード主体の高速移動を実現。

## 4. 技術仕様 (Technical Specifications)

### 4.1 開発環境
* **Manifest Version**: Manifest V3
* **Language**: JavaScript (ES6+), HTML5, CSS3 (Tailwind CSS推奨)
* **API**: `chrome.tabs`, `chrome.storage`, `chrome.alarms`, `chrome.runtime`
